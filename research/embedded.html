Embedded and real time system verification

Integrated systems are ubiquitous in the world we live in- mobile devices, biomedical, automobile, security, missile, aeronautics systems and many others are all heterogeneous in composition and versatile in functionality. System level design is frequently described at the transaction level, where hardware, software and control components can be integrated on a single platform. The design issues are compounded at this higher level. Verification, performance analysis, reliability etc of the entire system is daunting due to the complexity of these systems. Algorithms and tools to assist design are few and far between at this level. Our research is aimed at reliability, verification and performance/power analysis at the “system” level. We can generate functional assertions at the system level by mining transactions and guiding them with structural analysis. This provides system level design insights. Since performance and power are very important design parameters at system level, we generate performance assertions. We have developed powerful techniques for diagnosing performance violations and root causing the problem from transaction trace files that are 7-10 GB in size. In systems where the system level design is used as an executable specification for the RTL design, we have researched methods to perform sequential equivalence checking. These include assertion based equivalence checking between system level and RTL design. We are generating constraints in hard real time systems using Bayesian inferencing and formal methods, to be used for checking reliability and security of these systems. And so what…

The diagnosis of root causes of performance failures was used in Future Wei Technologies for their wireless baseband systems. Transaction trace files are massive, ranging between 7-10 GB in size. When there is a performance violation, diagnosing the problem and finding the root cause takes several engineers 2-4 weeks per violation. Our solution used static domain analysis strategically with an algorithm called concurrent pattern mining that we invented. As a result of this technology, the developers of the baseband wireless system were able to isolate 7 bugs in 2 hours. This methodology is now fully operational in Future-Wei technologies. Since we were not allowed access to internal proprietary software from the company, we developed a platform similar to the realistic platform used in the company over 5 months, and released it widely for research purposes. This is the first such open source AXI platform with a complex wireless baseband implementation that fills the void of lack of realistic testbeds in the system level research area. We are working with General Motors and Boeing Corporation for analyzing performance and security of real time systems.
